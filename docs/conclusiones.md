# Conclusiones de la tarea

Mediante el desarrollo y realización de esta actividad hemos podido integrar varios conceptos claves de esta unidad.

### Integración del flujo de trabajo moderno

La práctica ha permitido comprender cómo integrar distintas herramientas fundamentales en el desarrollo actual: GitHub para el control de versiones, GitHub Pages para el despliegue de contenido estático y Docker para la
contenedorización. Esto demuestra cómo un proyecto puede pasar del código fuente a un entorno desplegable de forma coherente y profesional.

### Despliegue sencillo y automatizado con GitHub Pages

Configurar el repositorio para servir una web estática a través de GitHub Pages ha evidenciado lo sencillo que es publicar contenido directamente desde GitHub sin necesidad de infraestructura adicional. Esto facilita compartir
documentación, portfolios o demos de forma inmediata.

### Portabilidad y consistencia mediante Docker

La creación del contenedor Docker ha mostrado la importancia de la portabilidad en entornos de despliegue. Al encapsular la web estática dentro de un contenedor, se garantiza que pueda ejecutarse de forma idéntica en cualquier 
sistema, evitando problemas derivados de diferencias entre entornos.

### Comprensión del ciclo completo de desarrollo y despliegue

Gracias a la combinación de GitHub y Docker, se ha trabajado un flujo completo: creación del código, versionado, publicación en un entorno remoto y puesta en marcha mediante un contenedor. Esto refleja prácticas reales de DevOps y 
mejora la comprensión de procesos de despliegue modernos.

### Mejora de habilidades técnicas clave

A través de esta tarea se han reforzado conocimientos esenciales: manejo de repositorios Git, configuración de GitHub Pages, creación de Dockerfiles, construcción de imágenes, ejecución de contenedores y comprensión del concepto 
de web estática.

### Base sólida para proyectos más avanzados

Como es de esperar a lo largo del curso seguramente realizaremos muchas actividades y tareas en las que deberemos emplear nuevamente estas herramientas que hemos visto como GitHub y Docker, en los que hemos desarrollado cierta 
soltura a la hora de trabajar con ellas.
